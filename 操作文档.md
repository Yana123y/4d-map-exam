- 每次随机抽取 **20 道题**

- 判断题选错不得分

## 2. 考生操作指南

### 2.1 进入考试

```
┌─────────────────────────────────────────┐
│  ① 打开网站链接                         │
│  ② 在输入框中填写你的姓名               │
│  ③ 点击 "开始答题" 按钮                 │
└─────────────────────────────────────────┘
```

> ⚠️ **姓名不能为空**，否则无法开始考试。姓名会记录在考试成绩中。

### 2.2 答题操作

| 操作 | 方式 |
|------|------|
| 选择答案 | 点击选项即可（单选/多选） |
| 切换题目 | 点击「上一题」「下一题」按钮 |
| 键盘快捷键 | `←` `→` 键切换题目 |
| 进度查看 | 顶部进度条显示已答题数 |

**三种题型标识：**

| 标签颜色 | 题型 | 说明 |
|----------|------|------|
| 🔵 蓝色 `单选题` | 单选题 | 点击一个选项即选中 |
| 🟡 黄色 `多选题` | 多选题 | 可点击多个选项，再次点击取消 |
| 🟢 绿色 `判断题` | 判断题 | 点击「正确」或「错误」 |

### 2.3 提交答卷

- 答完 20 道题后，最后一题页面会出现 **「提交答卷」** 按钮
- 如果还有题目未作答，按钮显示为灰色 `⚠️ 还有题目未作答`
- 提交前会弹出确认提示，确认后不可修改
- 也可使用 `Ctrl + Enter` 快捷键提交

### 2.4 查看结果

提交后显示：
- ✅ 总得分（百分制）
- 正确 / 错误题数
- 正确率
- 点击 **「查看答案解析」** 可逐题复盘
- 点击 **「重新抽题」** 重新开始新考试

**答案解析模式：**

| 颜色 | 含义 |
|------|------|
| 🟢 绿色边框 | 正确答案 |
| 🔴 红色边框 | 你选的错误答案 |
| 底部说明框 | 显示正确答案和你的答案对比 |

## 3. 管理后台操作指南

### 3.1 进入后台

```
考试页面右上角 → 点击 "⚙️ 管理" → 输入密码 → 登录
```

| 项目 | 值 |
|------|-----|
| 默认密码 | `admin123` |
| 修改密码 | 需修改源代码中 `ADMIN_PASSWORD` 变量 |

登录成功后进入管理后台，左侧为导航菜单。

### 3.2 📈 数据概览

首屏仪表盘，包含：

| 指标卡片 | 含义 |
|----------|------|
| 考试总次数 | 所有考生的考试次数合计 |
| 平均成绩 | 所有考试的平均分 |
| 考生人数 | 去重后的考生数量 |
| 及格人数(≥60) | 成绩 ≥ 60 分的次数 |
| 优秀人数(≥90) | 成绩 ≥ 90 分的次数 |
| 今日考试 | 当天考试场次 |

**成绩分布图**：横向柱状图，展示 90-100、80-89、70-79、60-69、0-59 各分数段人数。

**最近考试记录**：表格显示最近 10 条考试记录，可点击「查看」进入详情。

### 3.3 📋 考试记录

功能列表：

```
┌──────────────────────────────────────────────┐
│  🔍 搜索框   输入姓名实时筛选                │
│  📥 导出CSV  导出 Excel 可打开的表格文件      │
│  👁 查看详情 查看本次考试每道题的答题情况     │
│  🗑 删除     删除单条考试记录                 │
└──────────────────────────────────────────────┘
```

**查看详情弹窗**包含：
- 考生姓名、考试时间
- 成绩、正确/错误数
- 每道题的答题明细（正确答案 vs 你的答案）

### 3.4 👥 考生统计

按平均分从高到低排列所有考生：

| 列 | 说明 |
|----|------|
| 排名 | 按平均分排序 |
| 姓名 | 考生姓名 |
| 考试次数 | 该考生的总考试次数 |
| 最高分 / 最低分 | 该考生的最佳和最差成绩 |
| 平均分 | 综合平均分 |
| 趋势 | 📈 进步 / 📉 退步（最近两次对比） |

点击「历史」可查看该考生所有考试记录。

### 3.5 ❌ 错题分析

按错误次数排序的错题排行榜：

| 列 | 说明 |
|----|------|
| 题目 | 题目内容摘要 |
| 正确答案 | 标准答案 |
| 错误次数 | 该题被答错的累计次数 |
| 错误率 | 错误次数 / 总考试次数 |
| 出错考生 | 答错过该题的考生名单 |

> 高频错题 = 知识点薄弱环节，可用于针对性培训。

### 3.6 💾 数据管理

#### 导出数据

| 格式 | 用途 |
|------|------|
| **JSON** | 含完整答题记录，可用于备份或迁移到其他设备 |
| **CSV** | Excel/WPS 可直接打开，方便打印和汇报 |
| **打印** | 直接打印当前报表 |

#### 导入数据

```
操作步骤：
  ① 点击「选择文件」
  ② 选择之前导出的 .json 文件
  ③ 自动合并，跳过已存在的重复记录
  ④ 提示导入结果
```

> 📌 多人使用不同设备考试时，可将各设备数据导出后，在一个设备上统一导入合并。

#### 清空数据

```
⚠️ 危险操作：删除所有考试数据，不可撤销。
操作前建议先导出备份。
需要两次确认才会执行。
```

## 4. 数据管理

### 4.1 数据存储位置

考试数据存储在浏览器的 **localStorage** 中，这意味着：

- 每台电脑/手机的数据是独立的
- 清除浏览器缓存会导致数据丢失
- 建议定期导出备份

### 4.2 建议的数据管理流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 考生A考试 │    │ 考生B考试 │    │ 考生C考试 │
│ (设备1)   │    │ (设备2)   │    │ (设备3)   │
└────┬─────┘    └────┬─────┘    └────┬─────┘
     │ 导出JSON      │ 导出JSON      │ 导出JSON
     ▼               ▼               ▼
  ┌─────────────────────────────────────────┐
  │           管理员电脑统一导入             │
  │        查看全部数据、生成报表            │
  └─────────────────────────────────────────┘
```

### 4.3 考试结束后建议操作

1. 进入 **数据管理** → 导出 JSON 备份
2. 进入 **数据概览** → 截图留存总体情况
3. 进入 **错题分析** → 整理高频错题，针对性讲解
4. 进入 **考生统计** → 了解每位考生掌握程度

## 5. 常见问题

### Q1：换设备后数据没了？

> 数据存储在浏览器本地，换设备需要导出→导入。在旧设备上进入管理后台 → 数据管理 → 导出 JSON，在新设备上导入。

### Q2：如何修改管理密码？

> 需要修改源代码中第约 550 行的 `const ADMIN_PASSWORD = 'admin123';`，改为你的密码，然后重新部署。

### Q3：能否增加/修改题目？

> 可以。GitHub 仓库 `Yana123y/4d-map-exam` 中编辑 `index.html`，找到 `questionBank` 数组，按格式添加或修改题目，提交后自动部署。

**题目格式：**

```javascript
// 单选题
{id:101, type:"single", source:"来源", question:"题目文字？",
 options:["选项A","选项B","选项C","选项D"], answer:0},  // answer = 正确选项索引

// 多选题
{id:102, type:"multi", source:"来源", question:"题目（多选）",
 options:["选项A","选项B","选项C","选项D","选项E"],
 answer:[0,1,2]},  // answer = 所有正确选项索引数组

// 判断题
{id:103, type:"judge", source:"来源", question:"题目陈述。",
 answer:true},  // true=正确, false=错误
```

### Q4：手机能用吗？

> 可以。网站已适配手机端，直接浏览器打开即可。手机端管理后台侧边栏会自动隐藏，功能完整可用。

### Q5：能否多人同时在线考试？

> 可以。每个考生用自己的设备打开网站，输入自己的姓名考试。考试成绩各自保存在本地。

### Q6：GitHub Pages 访问慢怎么办？

> 国内访问 GitHub Pages 偶尔较慢。替代方案：将 `index.html` 文件保存到本地直接双击打开，或在公司内网服务器上托管。
